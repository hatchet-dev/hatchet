# Human-in-the-Loop

Human-in-the-loop workflows pause for human review or approval before continuing. An AI agent proposes an action, a human approves or rejects it, and the workflow resumes with the decision. [Durable Execution](/concepts/durable-workflows/durable-task-execution) lets the task pause without holding a worker slot. When the human responds, the task picks up exactly where it left off.

<HumanInLoopDiagram />

This builds on the [Long Waits](/concepts/durable-workflows/durable-task-execution/durable-sleep) pattern. The task calls [WaitForEvent](/concepts/durable-workflows/durable-task-execution/durable-events) and the worker slot is freed until the approval arrives. If you have a fixed DAG where the approval gate is known upfront, [Event Conditions](/concepts/durable-workflows/directed-acyclic-graphs/event-conditions) are a simpler alternative.

## Common patterns

| Pattern                         | Description                                                             |
| ------------------------------- | ----------------------------------------------------------------------- |
| **Content moderation**          | Agent flags content; human approves or rejects before publish           |
| **Financial approvals**         | Agent proposes payment or transfer; human approves via dashboard        |
| **Customer support escalation** | Agent drafts response; human reviews and sends, or edits before sending |
| **LLM output review**           | Agent generates copy or code; human approves before it goes live        |

For DAG workflows with a fixed approval gate, use [Event Conditions](/concepts/durable-workflows/directed-acyclic-graphs/event-conditions). For agent loops where the decision to wait is dynamic, use a durable task with `WaitForEvent`.
