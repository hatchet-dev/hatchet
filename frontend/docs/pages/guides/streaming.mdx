# Streaming / Live Updates

Streaming lets you push intermediate results from running tasks to clients in real-time: LLM token streams, progress updates, log lines, or live data feeds. Hatchet's streaming API connects workers to frontends without custom WebSocket infrastructure.

All you need is `put_stream()` in your worker and `subscribe_to_stream()` on the client. No separate WebSocket server required. See [Streaming](/concepts/streaming) for the full API.

## Step-by-step walkthrough

By the end of this guide you will have a task that emits chunks and a client that subscribes to receive them in real-time.

<Callout type="warning">
  Use **CANCEL_IN_PROGRESS** concurrency when streaming chat or agent responses.
  This ensures that when a user sends a new message, the previous streaming task
  is cancelled and a fresh one starts.
</Callout>

## Common Patterns

| Pattern                     | Description                                                                   |
| --------------------------- | ----------------------------------------------------------------------------- |
| **LLM token streaming**     | Stream generated tokens to a chat UI as the model produces them               |
| **Progress reporting**      | Emit percentage-complete updates for long-running batch or processing jobs    |
| **Live log tailing**        | Stream task log lines to a monitoring dashboard                               |
| **Real-time notifications** | Push status changes to connected clients as tasks progress through a pipeline |
| **Collaborative editing**   | Stream document changes between participants through Hatchet tasks            |
