# RAG & Data Indexing

RAG and indexing pipelines share a common shape: ingest documents, split them into chunks, generate embeddings, and write to a vector database. This guide walks through structuring these pipelines as Hatchet workflows.

<RAGPipelineDiagram />

Since the stages (ingest, chunk, embed, index) are fixed, RAG pipelines map directly to a [Pre-Determined Pipeline](/concepts/durable-workflows/directed-acyclic-graphs). You can add [Fanout](/concepts/durable-workflows/directed-acyclic-graphs/child-spawning) within the chunking stage to process documents in parallel.

## Multi-Tenant Indexing

For SaaS applications where multiple tenants share the same pipeline:

- **GROUP_ROUND_ROBIN concurrency** distributes scheduling fairly so no single tenant monopolizes workers
- **Additional metadata** tags each run with a tenant ID for filtering in the dashboard
- **Priority queues** allow higher-priority indexing jobs to run ahead of lower-priority ones
