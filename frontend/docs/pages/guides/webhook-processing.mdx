# Webhook Processing

Webhooks are how external services (Stripe, GitHub, Twilio, Shopify) push data to your application. Processing them reliably requires idempotency, retry handling, and spike protection. Hatchet turns incoming webhooks into durable tasks with automatic retries, concurrency control, and full observability.

There are two ways to get webhooks into Hatchet: point the external service at a Hatchet webhook URL directly (see [Webhooks](/concepts/webhooks) for setup), or receive the POST in your own HTTP handler (Express, FastAPI, etc.), verify the signature, and call `hatchet.event.push("webhook-key", payload)`.

## Common Patterns

| Pattern                    | Description                                                             |
| -------------------------- | ----------------------------------------------------------------------- |
| **Payment processing**     | Receive Stripe webhooks for payments, refunds, and subscription changes |
| **CI/CD triggers**         | React to GitHub push, PR, or deployment events                          |
| **Communication handling** | Process Twilio SMS/voice webhooks for customer support workflows        |
| **E-commerce events**      | Handle Shopify order, inventory, and fulfillment webhooks               |
| **Infrastructure alerts**  | Process PagerDuty, Datadog, or CloudWatch alarm webhooks                |
