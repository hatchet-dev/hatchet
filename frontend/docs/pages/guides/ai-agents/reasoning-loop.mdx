# Reasoning Loop

AI agents follow a **reason-act-observe** loop that can run for minutes or hours. This guide walks through structuring agent workflows in Hatchet so they survive interruptions, stream output to users, and don't exhaust resources.

<AgentLoopDiagram />

Under the hood, the agent loop is a [Cycle](/concepts/durable-workflows/durable-task-execution/child-spawning). Each iteration spawns a new child run via `RunChild`, so completed iterations survive crashes and worker slots are freed between iterations thanks to [Durable Execution](/concepts/durable-workflows/durable-task-execution).
