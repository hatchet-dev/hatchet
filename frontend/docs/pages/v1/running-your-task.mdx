# Running Tasks

Tasks and workflows in Hatchet can be invoked in a number of ways.

## Run and wait

Invoke a task and block until it completes and returns a result. Use this for synchronous workflows like fanout, LLM calls, or any time the task's output is needed before continuing.

<UniversalTabs items={["Python", "Typescript", "Go", "Ruby"]}>
  <Tabs.Tab title="Python">

The `Task` and `Workflow` objects can be used to invoke a task run by calling the `run` or `aio_run` methods. These method will block until the task or workflow completes and then will return the result.

<Snippet src={snippets.python.child.trigger.running_a_task} />

The run methods in Hatchet's Python SDK also have async flavors, which can be `await`ed. These are prefixed with `aio_`, such as `aio_run`.

<Snippet src={snippets.python.child.trigger.running_a_task_aio} />

Note that the type of `input` here is a Pydantic model that matches the input schema of the task or workflow being triggered.

  </Tabs.Tab>
  <Tabs.Tab title="Typescript">

The `Task` object can be used to invoke a task run by calling the `run` method. This method will return a promise that resolves when the task completes and returns the result.

<Snippet src={snippets.typescript.simple.run.running_a_task} />

  </Tabs.Tab>
  <Tabs.Tab title="Go">

The `Task` object can be used to invoke a task run by calling the `Run` function. This will block until the task completes and return the result.

### Running tasks in parallel

<UniversalTabs items={["Python", "Typescript", "Go", "Ruby"]} variant="hidden">
<Tabs.Tab title="Python">

<Snippet src={snippets.python.child.trigger.running_multiple_tasks} />

</Tabs.Tab>
<Tabs.Tab title="Typescript">

</Tabs.Tab>
<Tabs.Tab title="Go">

<Snippet src={snippets.go.simple.main.running_multiple_tasks} />

</Tabs.Tab>
  <Tabs.Tab title="Ruby">

In Ruby, multiple tasks can be spawned in parallel using `run_many`.

    <Snippet src={snippets.ruby.child.trigger.running_multiple_tasks} />

  </Tabs.Tab>
</UniversalTabs>

<Callout type="info">
  While you can run multiple tasks in parallel using the `Run` method, this is
  not recommended for large numbers of tasks. Instead, use [bulk run
  methods](/v1/bulk-run) for large parallel task execution.
</Callout>

## Fire and forget

Fire-and-forget-style triggering enqueues a task without waiting for the result. This is useful for background jobs like sending emails, processing uploads, or kicking off long-running pipelines.
