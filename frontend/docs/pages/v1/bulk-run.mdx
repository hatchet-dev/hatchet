# Bulk Run Many Tasks

Running a task multiple times with different inputs incurs per-call network overhead. The bulk run methods dispatch multiple runs in a single request, reducing that overhead.

<UniversalTabs items={["Python", "Typescript", "Go", "Ruby"]}>
  <Tabs.Tab title="Python">

Use `aio_run_many` to dispatch multiple runs and receive a list of results.

<Snippet src={snippets.python.child.bulk.bulk_run_a_task} />

<Callout type="info">
  `Workflow.create_bulk_run_item` is a typed helper to create the inputs for
  each task.
</Callout>

Available bulk methods on the `Workflow` object:

- `aio_run_many`
- `aio_run_many_no_wait`

Blocking variants:

- `run_many`
- `run_many_no_wait`

Bulk methods called from within a task associate the child runs with the parent in the dashboard.

  </Tabs.Tab>
  <Tabs.Tab title="Typescript">

Pass an array of inputs to the `run` method to dispatch multiple runs and receive a list of results.

<Snippet src={snippets.typescript.simple.bulk.bulk_run_a_task} />

Available bulk methods on the `Task` object:

- `run`
- `runNoWait`

Bulk methods called from within a task via the context parameter associate the child runs with the parent in the dashboard.

Available bulk methods on the `Context` object are: `bulkRunChildren`, `bulkRunChildrenNoWait`.
