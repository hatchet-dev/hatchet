# Recurring Runs with Cron

[Cron](https://en.wikipedia.org/wiki/Cron) is a time-based job scheduler. In Hatchet, cron triggers define when a task executes automatically on a recurring schedule.

Common use cases include:

1. Running a daily report at a specific time.
2. Sending weekly digest emails summarizing user activity.
3. Running a monthly billing process to generate invoices.

Hatchet supports two ways to define cron triggers:

- [Task Definitions](/v1/cron-runs#defining-a-cron-in-your-task-definition): A cron expression is embedded directly in the task definition.
- [Programmatic API](/v1/cron-runs#programmatically-creating-cron-triggers): The Hatchet SDK is used to create and manage cron triggers at runtime.

<Callout type="warning">
  The cron expression controls when Hatchet **enqueues** the task, not when the
  run starts. Concurrency limits, rate limits, and retry policies can affect the
  actual start time.
</Callout>

### Cron Expression Syntax

Hatchet supports standard cron syntax with 5-field and 6-field expressions. Fields are space-separated. A 6-field expression includes an optional leading seconds field.

```
┌───────────── second (0 - 59) (optional)
| ┌───────────── minute (0 - 59)
│ │ ┌───────────── hour (0 - 23)
│ │ │ ┌───────────── day of the month (1 - 31)
│ │ │ │ ┌───────────── month (1 - 12)
│ │ │ │ │ ┌───────────── day of the week (0 - 6) (Sunday to Saturday)
* * * * * *
```

Use a service such as https://crontab.guru to validate cron expressions.

<Callout type="info">
  Hatchet Cloud meters by task run. Use sub-minute schedules (seconds field)
  carefully. For pricing questions, [contact
  us](https://hatchet.run/office-hours).
</Callout>

## Considerations

1. **Time zone**: All cron schedules run in UTC. Account for time zone offsets when writing cron expressions.

2. **Execution time**: Hatchet enqueues tasks as close to the scheduled time as possible, but minor delays may occur due to system load.

3. **Missed schedules**: Hatchet does not replay missed executions after downtime. The next trigger fires at the next scheduled time.

4. **Overlapping runs**: If a prior run is still active when the next scheduled time arrives, Hatchet starts a new instance or applies the configured [concurrency](/v1/concurrency) policy.
