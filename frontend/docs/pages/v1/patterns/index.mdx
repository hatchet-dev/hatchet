# Workflow Design

There are two ways to build durable workflows in Hatchet: **DAGs** and **durable tasks**.

The short version: if you know every step your workflow will take before it starts, use a DAG. If the shape of your work is decided at runtime — based on inputs, model responses, or intermediate results — use a durable task. You can freely mix both in the same application.

- [**Durable Tasks**](./durable-tasks) — Tasks that use durable execution primitives (waits + child spawning) with checkpointing and replay.
- [**DAG Workflows**](./directed-acyclic-graphs) — Fixed multi-step workflows where every task and dependency is declared upfront.
- [**Choosing an Approach**](./mixing-patterns) — Practical guidance on when to use each approach, and how to combine them.
